# 08 Delivery And Execution

## Workstreams
- Utility or district rule collection
- Registry normalization
- Page generation and templating
- Public request-help routing
- Analytics and index controls

## Milestone 1
- Baseline state foundation
- First utility or district pages
- First evergreen guides

## Milestone 2
- First representative expansion state
- Failed-test pages
- Approved-tester pages
- Public provider profiles where source quality supports them

## Milestone 3
- Metro discovery pages
- Stronger request-help coverage
- Clarity and conversion improvements on public owner-facing routes

## Acceptance criteria
- A new agent can build from this folder without chat history
- No local page is indexed without an official source
- Non-official directory content is never confused with official guidance
- Tester routing is measurable by page family

## Execution order
1. Rule and sourcing owner
2. Backend and templating owner
3. SEO and indexing owner
4. Public request-help and provider-directory owner
