# 05 Editorial Rules And Execution

## 1) Editorial rules
- Lead with the answer, not theory.
- Use exact utility naming throughout.
- Keep every local page tightly sourced and timestamped.
- Separate official requirements from non-official directory suggestions visually and verbally.
- Avoid jargon when the user is in a deadline or failure state.

## 2) Trust rules
- This is public-safety-adjacent content.
- Never imply government endorsement of providers.
- Never promise legal compliance.
- Always route to the official program page when possible.

## 3) Delivery phases
### Phase 1
- Build 4 evergreen guides
- Build registry schema and file contracts
- Build freshness audit and stale suppression
- Launch 10-15 utility pages in one state

### Phase 2
- Expand to 25-50 utility pages across two states only after one successful refresh cycle
- Add failed-test pages and either official approved-tester pages or non-official find-a-tester pages
- Add public provider profiles only where the source quality supports them

### Phase 3
- Add metro provider pages
- Add stronger request-help support
- Expand to more states only after refresh operations work

## 4) Success metrics
- Indexed utility pages
- Utility pages with valid sources
- Pilot pages refreshed within SLA
- Provider CTR from local pages
- Request-help completion rate
- Stale page rate

## 5) Definition of done for launch
- 10+ utility pages with official links and verification dates
- 4 evergreen guides published
- One repeatable source-verification workflow
- Stale suppression and freshness audit active
- Public directory placement rules defined
- Local page template stable enough for expansion
